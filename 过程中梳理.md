  - 时间追踪 Hook (hooks/useTimeTracking.ts:14) 与工具类 (utils/timeTracker.ts:18) 会直接连到默认 http://localhost:5004 的第二阶段接口，因此“VKS 入口→练习”这条路径已经能把学习

  问题结论
  当前仓库已经具备 1) 第二阶段后端（推荐/复习/统计）和 2) 前端时间追踪框架，但学习界面仍停留在“测试版”：静态词卡 + 静态练习，既未根据推荐动态切换，也未把复习计划嵌入整条学习路
  径。还缺少用户体系、多词汇数据、复习入口等论文中描述的完整闭环。

  建议的下一步方案

  1. 打通词汇数据管道：实现 src/lib/apiClient 上的 fetchJson，新建 useWordData(wordId) Hook，用于 /word/<id>，在词卡/搭配/例句页面读取真实内容；并把 wordId 通过入口页存入

  4. 扩充学习内容与复习流程：补充数据库（或编写导入脚本）多词条数据；实现复习列表 UI，按 /api/review/user/.../due 展示并触发 SRS 流程；为练习页接入真实题目与判题接口。
  5. 完善用户与配置：引入简单登录/用户切换（可以先做伪账号选择），确保推荐、复习、统计都基于真实 user_id；补充 .env 与 README 说明如何配置 NEXT_PUBLIC_API_BASE_URL、如何启动阶
  段二服务。

  按以上步骤推进，可以把目前的“演示+静态页面”逐步演化成论文里描述的、由推荐驱动的完整学习 App。建议从步骤 1 开始，因为只要词卡与练习能读取真实数据，就能快速看见系统“活”起来；随
