# 交互功能测试指南

## 🧪 如何测试交互功能

### 1. 打开浏览器开发者工具
1. 访问 http://localhost:3000
2. 按 F12 或右键选择"检查元素"
3. 切换到 "Console" 标签页

### 2. 测试首页交互
**页面**: http://localhost:3000
**功能**: "开始学习" 按钮点击

**测试步骤**:
1. 点击绿色的"开始学习"按钮
2. 查看Console中是否显示: `Start Learning button clicked!`
3. 页面应该跳转到词汇学习入口页面

**预期结果**: 
- Console有日志输出
- 页面成功跳转

### 3. 测试词汇学习入口页面
**页面**: http://localhost:3000/word-learning-entrance
**功能**: 词汇掌握程度选择和路由跳转

**测试步骤**:
1. 尝试点击任一单选项 (A、B、C、D、E)
2. 查看Console中是否显示: `Option selected: [选择的选项]`
3. 点击"CONTINUE"按钮
4. 查看Console中的跳转日志

**预期结果**: 
- 单选框可以正常选择
- Console显示选择日志
- 点击CONTINUE有相应反馈

### 4. 测试练习页面
**页面**: http://localhost:3000/exercise
**功能**: 答题和反馈

**测试步骤**:
1. 选择一个答案选项
2. 点击"提交答案"按钮
3. 查看反馈信息
4. 点击"继续"进入下一题

**预期结果**: 
- 能正常选择答案
- 显示正确/错误反馈
- 能进入下一题

### 5. 交互调试测试页面
**页面**: http://localhost:3000/debug-test
**功能**: 基础交互功能验证

**测试步骤**:
1. 点击"测试点击事件"按钮
2. 观察计数是否增加
3. 点击"测试路由跳转"按钮
4. 验证是否跳转成功

## 🔍 常见问题排查

### 问题1: 按钮点击没有反应
**可能原因**:
- JavaScript未加载
- 事件监听器未绑定
- CSS样式阻止了点击

**排查方法**:
1. 检查Console是否有JavaScript错误
2. 验证按钮的onclick事件是否正确绑定
3. 检查CSS是否有 `pointer-events: none`

### 问题2: 页面跳转失败
**可能原因**:
- 路由配置错误
- useRouter钩子问题
- 目标页面不存在

**排查方法**:
1. 检查路由路径是否正确
2. 验证目标页面文件是否存在
3. 查看Next.js的路由配置

### 问题3: 表单选择不工作
**可能原因**:
- 状态管理问题
- 组件props传递错误
- Radix UI组件配置问题

**排查方法**:
1. 检查useState是否正确更新
2. 验证组件的value和onValueChange属性
3. 查看组件的受控/非受控状态
