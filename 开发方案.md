# 单词学习系统开发方案

## 项目概览

### 项目描述
基于PDF论文中的原型图开发的中文单词学习系统，采用前后端分离架构，提供多维度的单词学习功能。

### 技术栈
- **前端**: Next.js + React + TypeScript + Tailwind CSS + shadcn/ui
- **后端**: Flask + SQLAlchemy + SQLite
- **开发环境**: conda zhipu环境

## 当前项目状态分析

### 已完成功能
1. **后端API基础框架**
   - Flask应用基础结构
   - SQLite数据库模型设计（Word, Example, Collocation, Character）
   - 基础API接口（获取单词、提交练习、单词列表）
   - 示例数据初始化

2. **前端页面结构**
   - Next.js项目配置
   - UI组件库集成（shadcn/ui）
   - 移动端界面设计（iPhone样式）
   - 多个学习页面组件

3. **已实现的页面**
   - 首页 (`index.tsx`)
   - 单词学习主页面 (`word-learning.tsx`)
   - 字符学习页面 (`character-learning.tsx`) 
   - 例句学习页面 (`sentence-learning.tsx`)
   - 搭配学习页面 (`collocation-learning.tsx`)
   - 入门评估页面 (`word-learning-entrance.tsx`)
   - 练习页面 (`exercise.tsx`)

### 需要完善的功能

基于论文分析，系统需要实现以下核心功能：

#### 论文中的系统设计要求：
1. **词汇学习入口（VKS测试改编）**
   - A: 从没见过这个词 → 从字开始学习
   - B: 不知道意思 → 从词开始学习  
   - C: 知道意思，不知道搭配 → 从搭配开始学习
   - D: 知道搭配，不知道用法 → 从句子开始学习
   - E: 可以在句子中使用 → 直接进入练习

2. **学习内容设计**
   - 字词音形义（拼音、字形、词形、高频义项）
   - 词汇搭配（与HSK 1-3级词汇的搭配，标注等级）
   - 例句学习（低难度例句，高级词汇用括号标明）
   - 翻译辅助（字、词、搭配、例句的英文翻译）
   - 易混淆词分开学习

3. **练习模块题型**
   - 词汇释义题（英文释义选择中文词汇）
   - 词汇搭配题（每词两个搭配，一错全错时都给反馈）
   - 词汇填词题（手写输入功能）

4. **自适应复习模块**
   - 训练组：按自适应规则复习
   - 测试组：按算法推荐复习
   - 基于练习结果推荐复习内容

5. **测试模块**
   - 词汇释义题
   - 词汇搭配题
   - 易混淆词辨析题  
   - 词汇填词题

## 第二阶段数据库扩展

### 新增分析表
```sql
-- 用户学习模型表
CREATE TABLE user_learning_model (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id VARCHAR(50) NOT NULL,
    model_type VARCHAR(20) NOT NULL,  -- 'memory', 'preference', 'difficulty'
    model_data TEXT NOT NULL,         -- JSON格式的模型参数
    confidence_score REAL DEFAULT 0.0,
    last_updated DATETIME DEFAULT CURRENT_TIMESTAMP,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 复习调度表
CREATE TABLE review_schedule (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id VARCHAR(50) NOT NULL,
    word_id INTEGER NOT NULL,
    scheduled_time DATETIME NOT NULL,
    priority_score REAL DEFAULT 0.0,
    review_type VARCHAR(20) DEFAULT 'normal',  -- 'normal', 'urgent', 'maintenance'
    algorithm_version VARCHAR(20),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    completed_at DATETIME,
    
    FOREIGN KEY (word_id) REFERENCES word (id),
    INDEX idx_user_schedule (user_id, scheduled_time),
    INDEX idx_priority (priority_score DESC)
);

-- 学习模式分析表
CREATE TABLE learning_pattern_analysis (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id VARCHAR(50) NOT NULL,
    analysis_type VARCHAR(30) NOT NULL,  -- 'time_pattern', 'error_pattern', 'efficiency'
    pattern_data TEXT NOT NULL,          -- JSON格式的模式数据
    analysis_date DATE NOT NULL,
    confidence_level REAL DEFAULT 0.0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 第二阶段技术风险和应对

### 算法复杂性风险
- **风险**: 自适应算法过于复杂，难以调试和优化
- **应对**: 采用模块化设计，每个算法独立开发和测试

### 数据质量风险
- **风险**: 测试数据不够真实，影响算法效果
- **应对**: 构造多样化的真实场景测试数据

### 性能风险
- **风险**: 复杂推荐计算影响系统响应速度
- **应对**: 异步计算、缓存策略、性能监控

### 用户接受度风险
- **风险**: 推荐功能用户接受度不高
- **应对**: 用户研究、A/B测试、渐进式功能发布
