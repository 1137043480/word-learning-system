## 📊 项目概览

### 项目描述
基于PDF论文《自适应对外汉语中级词汇学习系统研究与设计》开发的中文词汇学习系统，采用前后端分离架构，提供多维度的词汇学习功能。

### 技术栈
- **前端**: Next.js 14 + React + TypeScript + Tailwind CSS + shadcn/ui
- **后端**: Flask + SQLAlchemy + SQLite
- **开发环境**: conda zhipu环境 (Python 3.11.9)
- **操作系统**: macOS 21.6.0

### 核心特性
- VKS测试适配的学习入口
- 字符→词汇→搭配→句子的完整学习链
- 三种练习题型（释义、搭配、填词）
- 自适应推荐引擎
- 间隔重复算法
- 学习数据分析Dashboard
- 毫秒级时间追踪

### 技术选型理由

#### 前端技术
- **Next.js 14**: 
  - 服务端渲染提升首屏加载速度
  - 文件路由系统简化开发
  - 良好的TypeScript支持
  
- **TypeScript**: 
  - 类型安全减少运行时错误
  - 更好的IDE支持和代码提示
  - 便于大型项目维护

- **Tailwind CSS**: 
  - 原子化CSS提高开发效率
  - 响应式设计简单直观
  - 体积小性能好

- **shadcn/ui**: 
  - 基于Radix UI的高质量组件
  - 完全可定制
  - 无缝集成Tailwind

#### 后端技术
- **Flask**: 
  - 轻量级灵活
  - 易于快速原型开发
  - 丰富的扩展生态

- **SQLAlchemy**: 
  - 强大的ORM功能
  - 支持多种数据库
  - 优雅的查询API

- **SQLite**: 
  - 无需额外服务器
  - 适合中小规模数据
  - 易于备份和迁移

## 📐 开发规范

### 项目结构与模块组织

Next.js前端位于 `pages/` 目录，每个学习模式作为独立路由（如 `pages/word-learning.tsx` 和 `pages/time-tracking-demo.tsx`）。基于shadcn的可复用UI组件在 `components/ui/` 中，共享的TypeScript工具函数在 `src/lib/` 中。Python服务通过Flask应用提供自适应逻辑，如 `app_phase2.py` 和数据加载器 `simple_test_data.py`；数据库文件保存在SQLite文件 `words.db` 和 `words_extended.db` 中。Shell脚本（`start_frontend.sh`、`start_system.sh`）用于常见启动流程。

### 构建、测试和开发命令

```bash
# 安装依赖
npm install  # 前端依赖
pip install flask flask-cors flask-sqlalchemy  # 后端依赖

# 开发环境
npm run dev  # 启动前端开发服务器 (http://localhost:3000)
python app_phase2.py  # 启动后端API (http://localhost:5004)
bash start_system.sh  # 一键启动系统

# 生产环境
npm run build  # 构建生产版本
npm run start  # 运行生产服务器

# 测试
node test_time_tracking.js  # API测试
npx eslint .  # 代码检查
```

### 代码风格与命名约定

#### 前端代码规范
- **React组件**: 使用函数式组件 + TypeScript
- **文件命名**: PascalCase for components (`LearningDashboard.tsx`)
- **缩进**: 2空格
- **Tailwind类名**: 按布局→间距→颜色分组
- **className合并**: 使用 `src/lib/cn` 工具函数

**示例**:
```typescript
// ✅ 好的例子

interface ButtonProps {
  variant?: "primary" | "secondary"
  children: React.ReactNode
}

export const Button = ({ variant = "primary", children }: ButtonProps) => {
  return (
    <button 
      className={cn(
        "px-4 py-2 rounded-lg",  // 布局和间距
        "font-medium",            // 文本
        variant === "primary" 
          ? "bg-blue-500 text-white"  // 颜色
          : "bg-gray-200 text-gray-700"
      )}
    >
      {children}
    </button>
  )
}

// ❌ 不好的例子
```

## 💾 数据库设计

### 数据库架构

系统使用SQLite数据库，分为两个主要文件：
- `words.db` - 原始词汇数据
- `words_extended.db` - 扩展学习数据（包含测试数据）

### 核心数据表

#### 1. 词汇数据表

```sql
-- 词汇主表
CREATE TABLE word (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word VARCHAR(50) NOT NULL,
    pinyin VARCHAR(100),
    definition TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 字符表
CREATE TABLE character (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    character VARCHAR(10) NOT NULL,
    pinyin VARCHAR(50),
    definition TEXT,
    FOREIGN KEY (word_id) REFERENCES word (id)
);

-- 搭配表
CREATE TABLE collocation (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    collocation VARCHAR(200) NOT NULL,
    translation VARCHAR(200),
    FOREIGN KEY (word_id) REFERENCES word (id)
);

-- 例句表
CREATE TABLE example (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL,
    sentence TEXT NOT NULL,
    pinyin TEXT,
    translation TEXT,
    FOREIGN KEY (word_id) REFERENCES word (id)
);
```

#### 2. 用户数据表

```sql
-- 用户档案表
CREATE TABLE user_profile (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id VARCHAR(50) UNIQUE NOT NULL,
    username VARCHAR(100),
    language_level VARCHAR(20),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_active DATETIME
);

-- 用户进度表
CREATE TABLE user_progress (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id VARCHAR(50) NOT NULL,
    word_id INTEGER NOT NULL,
    mastery_level REAL DEFAULT 0.0,
    review_count INTEGER DEFAULT 0,
    correct_count INTEGER DEFAULT 0,
    last_reviewed DATETIME,
    next_review DATETIME,
    ease_factor REAL DEFAULT 2.5,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (word_id) REFERENCES word (id),
    UNIQUE (user_id, word_id)
);
```

#### 3. 学习数据表

```sql
-- 学习会话表
CREATE TABLE learning_session (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id VARCHAR(100) UNIQUE NOT NULL,
    user_id VARCHAR(50) NOT NULL,
    word_id INTEGER,
    session_type VARCHAR(20),
    module_type VARCHAR(50),
    start_time DATETIME NOT NULL,
    end_time DATETIME,
    duration_seconds INTEGER,
    active_time_seconds INTEGER,
    completed BOOLEAN DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (word_id) REFERENCES word (id)
);

-- 练习记录表
CREATE TABLE exercise_record (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id VARCHAR(100),
    user_id VARCHAR(50) NOT NULL,
    word_id INTEGER,
    question_id VARCHAR(100),
    question_type VARCHAR(50),
    user_answer TEXT,
    correct_answer TEXT,
    is_correct BOOLEAN,
    response_time_seconds REAL,
    hesitation_count INTEGER DEFAULT 0,
    confidence_level INTEGER,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (word_id) REFERENCES word (id),
    FOREIGN KEY (session_id) REFERENCES learning_session (session_id)
);

-- 学习事件表
CREATE TABLE learning_event (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id VARCHAR(100),
    user_id VARCHAR(50) NOT NULL,
    event_type VARCHAR(50) NOT NULL,
    event_target VARCHAR(100),
    event_data TEXT,
    timestamp DATETIME NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES learning_session (session_id)
);
```

#### 4. 推荐数据表

```sql
-- 自适应推荐表
CREATE TABLE adaptive_recommendation (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id VARCHAR(50) NOT NULL,
    word_id INTEGER NOT NULL,
    recommendation_type VARCHAR(50),
    module VARCHAR(50),
    priority_score REAL DEFAULT 0.0,
    confidence_score REAL DEFAULT 0.0,
    reason TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    accepted BOOLEAN,
    completed_at DATETIME,
    FOREIGN KEY (word_id) REFERENCES word (id)
);
```

### 数据库索引

为提升查询性能，创建以下索引：

```sql
-- 用户相关索引
CREATE INDEX idx_user_id ON user_progress(user_id);
CREATE INDEX idx_user_word ON user_progress(user_id, word_id);
CREATE INDEX idx_next_review ON user_progress(next_review);

-- 会话相关索引
CREATE INDEX idx_session_user ON learning_session(user_id, start_time);
CREATE INDEX idx_session_id ON exercise_record(session_id);

-- 推荐相关索引
CREATE INDEX idx_recommendation_user ON adaptive_recommendation(user_id, created_at);
CREATE INDEX idx_recommendation_priority ON adaptive_recommendation(priority_score DESC);
```
