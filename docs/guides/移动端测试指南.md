## 🔍 详细测试步骤

### 测试1: PWA安装

**iOS (Safari)**
```
1. 打开 Safari 访问应用
2. 点击分享按钮 (底部中间)
3. 向下滚动，选择"添加到主屏幕"
4. 确认添加
5. 从主屏幕打开应用
6. 验证独立窗口运行
```

**Android (Chrome)**
```
1. 打开 Chrome 访问应用
2. 点击右上角菜单 (三个点)
3. 选择"添加到主屏幕"或"安装应用"
4. 确认添加
5. 从主屏幕打开应用
6. 验证独立窗口运行
```

### 测试2: 离线功能

```
1. 正常访问应用，浏览几个页面
2. 开启飞行模式
3. 刷新页面
4. 验证已缓存页面可访问
5. 尝试访问未缓存页面
6. 验证显示离线提示
7. 关闭飞行模式
8. 验证自动恢复连接
```

### 测试3: 手写输入

```
1. 进入练习页面
2. 找到填空题
3. 测试 Canvas 手写
   - 绘制清晰度
   - 清除功能
   - 提交功能
4. 切换到键盘模式
5. 测试键盘输入
6. 验证答案匹配
```

### 测试4: 音频播放

```
1. 进入词学习页面
2. 点击播放按钮
3. 验证发音
4. 测试多个播放按钮
5. 验证状态更新
6. 测试错误处理
```

### 测试5: 触控手势

```
1. 测试滑动切换
2. 测试长按
3. 测试双击
4. 测试缩放(如果支持)
```

## 🐛 常见问题

### 问题1: iOS上音频不播放
**原因**: iOS需要用户交互才能播放  
**解决**: 确保音频由用户点击触发

### 问题2: Canvas手写在某些设备卡顿
**原因**: 绘图性能问题  
**解决**: 降低采样率，优化Canvas代码

### 问题3: PWA安装选项不显示
**原因**: 未满足PWA条件  
**解决**: 检查manifest.json和service worker

### 问题4: 横屏布局异常
**原因**: CSS未适配横屏  
**解决**: 添加横屏媒体查询

### 问题5: 输入框被键盘遮挡
**原因**: 视口高度计算问题  
**解决**: 使用vh单位和scrollIntoView

## 🔧 调试工具

### 桌面调试

**Chrome DevTools (推荐)**
```
1. 打开 Chrome DevTools (F12)
2. 点击设备图标 (Toggle device toolbar)
3. 选择要模拟的设备
4. 开始测试
```

**测试不同网络条件**
```
1. Network 标签
2. 选择 Throttling
3. 选择 Fast 3G / Slow 3G
4. 测试加载性能
```

### 移动端调试

**iOS (Safari)**
```
1. iPhone: 设置 > Safari > 高级 > 启用Web检查器
2. Mac: Safari > 偏好设置 > 高级 > 勾选"在菜单栏中显示开发"
3. 连接iPhone到Mac
4. Safari > 开发 > [你的iPhone] > 选择页面
```

**Android (Chrome)**
```
1. 手机: 设置 > 关于手机 > 连续点击版本号7次(启用开发者选项)
2. 设置 > 开发者选项 > 启用USB调试
3. 连接到电脑
4. Chrome: chrome://inspect
5. 选择设备和页面
```
