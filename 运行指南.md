# 🚀 自适应学习系统运行指南

## 📋 快速启动

### 1. 启动前端服务
```bash
# 如果前端服务未运行
npm run dev
```
前端将在 http://localhost:3000 启动

### 2. 启动第二阶段API服务
```bash
# 方法1: 直接启动
python app_phase2.py

# 方法2: 使用启动脚本 (推荐)
./start_system.sh
```
第二阶段API将在 http://localhost:5004 启动

### 3. 验证系统运行
打开浏览器访问：http://localhost:3000/system-status

## 👤 用户与环境配置

- 页面右下角的「学习控制台」支持在多个测试账号之间快速切换（默认提供 `test_user_001` 等账号），也可以输入自定义 `user_id` 以验证个性化数据流。
- 控制台会自动同步后台 `GET /api/users` 返回的账号列表，可点击“刷新”按钮从数据库即时加载最新测试用户；手动输入的账号会被持久化，便于重复调试。
- 控制台同时允许运行期间修改 API Base URL，便于在不同环境之间跳转；如需长期固定，可在项目前端根目录创建 `.env.local`，写入 `NEXT_PUBLIC_API_BASE_URL=http://localhost:5004`。
- 每个账号的学习进度、推荐词汇与练习上下文都会存储在 `localStorage`（命名空间 `learningSession:<userId>`），切换账号时互不干扰；可通过“清除当前会话数据”按钮快速恢复默认状态。
- Dashboard 会在顶部展示当前 session 概览（词汇、模块、更新时间），并提供“一键继续学习”按钮及最近会话列表，便于快速跳转回对应页面。
- 控制台还会显示每个账号的最近学习词汇/模块与学习时间，数据由后端 `/api/users` 及本地 `learningSession:<userId>` 缓存共同驱动。

## 🔧 常见问题解决

### 1. 第二阶段API无法启动
**问题**: 运行 `python app_phase2.py` 没有输出或报错

**解决方案**:
```bash
# 检查Python环境
conda activate zhipu

# 检查依赖
pip install flask flask-cors flask-sqlalchemy

# 如果有端口冲突
sudo lsof -i :5004
# 杀死占用进程后重新启动
```

### 2. 前端页面无法访问新功能
**问题**: 访问新页面时出现404或编译错误

**解决方案**:
```bash
# 重新安装依赖
npm install recharts lucide-react

# 重启前端服务
npm run dev
```

### 3. API请求失败
**问题**: Dashboard或演示页面无法获取数据

**解决方案**:
1. 确认第二阶段API服务在运行 (http://localhost:5004)
2. 检查CORS设置
3. 查看浏览器控制台错误信息

### 4. 数据库相关问题
**问题**: 没有测试数据或数据库连接失败

**解决方案**:
```bash
# 生成测试数据
python simple_test_data.py

# 检查数据库文件
ls -la words_extended.db
```
